Propriedades Position e Display no CSS

display: block

Ocupa toda a largura disponível e força quebra de linha. Use quando quer estruturar a página em seções (ex: divs, parágrafos). Não é ideal quando você quer elementos lado a lado.

1
2

display: inline

Fica na mesma linha, mas NÃO aceita largura e altura. Use apenas para textos ou pequenos elementos (como span). Evite quando precisar controlar tamanho.

1
2

display: inline-block

Fica na mesma linha e permite definir largura e altura. Use quando quer alinhar elementos lado a lado, mas ainda controlar tamanho — diferente do inline.

1
2

display: flex

Cria um layout flexível em linha. Use quando precisa alinhar, centralizar ou distribuir elementos facilmente. Melhor escolha para layouts modernos simples (ex: menus, cards).

1
2

display: inline-flex

Igual ao flex, mas não quebra linha. Use quando quer flex dentro de um fluxo de texto ou junto com outros elementos inline.

1
2

display: inline-grid

Grid sem quebrar linha. Use quando precisa de grid, mas dentro de um fluxo inline.

1
2

display: grid

Cria layout em linhas e colunas. Use quando precisa organizar conteúdo em estrutura mais complexa (ex: páginas inteiras, dashboards). Melhor que flex quando há duas dimensões.

1
2

display: flow-root

Cria um novo contexto de layout e resolve problemas com float (como elementos "escapando" do container). Use quando tiver float e o container estiver colapsando.

flow-root (corrige)

display: table

Faz o elemento se comportar como tabela. Use raramente — geralmente flex ou grid são melhores, mas pode ser útil para compatibilidade ou layouts específicos.

1
2

display: inline flow-root

Combina comportamento inline com isolamento interno (flow-root). Use quando precisa evitar problemas de float sem quebrar linha.

display: block flow

É o comportamento padrão (equivalente ao block). O elemento ocupa toda a linha e organiza filhos no fluxo normal. Use quando não precisa de layout especial.

1
2

display: block flex

Equivale ao display: flex. O elemento ocupa a linha inteira (block) e organiza os filhos com flex. Use quando quer layout flexível ocupando toda a largura.

1
2

display: block grid

Equivale ao display: grid. Ocupa a linha inteira e organiza em grade. Use quando precisa de layout estruturado em colunas/linhas ocupando o espaço todo.

1
2

display: inline flow

Equivale ao inline. Fica na mesma linha e usa fluxo normal internamente. Use para textos ou elementos pequenos — não para layout.

1
2

position: static

É o comportamento padrão. O elemento segue o fluxo normal e ignora top/left/right/bottom. Use quando NÃO precisa controlar posição — evita complexidade desnecessária.

static

position: relative

Move o elemento sem tirá-lo do fluxo. O espaço original continua reservado. Use quando quer ajustar posição visual sem afetar outros elementos — ideal para pequenos deslocamentos.

relative

position: absolute

Tira o elemento do fluxo normal da página e permite posicioná-lo livremente dentro do seu elemento, dando controle total sobre sua posição.

absolute

position: fixed

Fica preso na tela (viewport), independente da rolagem. Use quando o elemento deve sempre estar visível (ex: botão de suporte, menu fixo). Evite em excesso pois pode atrapalhar UX.

fixed

position: sticky

Funciona como relative até atingir um limite (top, por exemplo), depois vira fixed. Use para menus que acompanham a rolagem sem ficar fixos o tempo todo.

sticky
